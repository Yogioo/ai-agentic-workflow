# AI Agentic Workflow 总览入口

## 1. 文档用途

本文档是整套 Agentic Workflow 的统一导航页。

它用于回答三个问题：

1. 这套流程由哪些文档组成
2. 每份文档分别解决什么问题
3. Main Agent 与 Sub Agent 在执行时应按什么顺序读取这些文档

---

## 2. 文档总览

### 2.1 框架文档

- `framework/MainAgent.md`：定义 Main Agent 的角色定位、职责边界、任务拆解要求、派单要求与异常处理方式
- `framework/DevSubAgent.md`：定义 Dev Sub Agent 的执行边界、输入要求、实现原则、回传要求与 Fail-Fast 规则
- `framework/QASubAgent.md`：定义 QA Sub Agent 的审查边界、测试原则、输出要求与阻塞处理方式
- `framework/Kanban任务模板.md`：定义当前阶段 Kanban 的标准结构、字段要求、状态定义与拆解规则
- `framework/SubAgent任务下发模板.md`：定义 Main Agent 向 Dev / QA Sub Agent 正式派单时的任务包格式，以及 Sub Agent 回传格式
- `framework/文档目录规范.md`：定义项目文档目录、命名规则、读取优先级、归档规则与文档边界
- `framework/命令体系规范.md`：定义默认命令入口、内部状态机和 `start` / `next` 的行为约定

### 2.2 运行模板文档

- `templates/bootstrap/当前PRD.md`：用于生成 `当前PRD.md`
- `templates/bootstrap/当前RoadMap.md`：用于生成 `当前RoadMap.md`
- `templates/bootstrap/BUG追踪.md`：用于生成 `BUG追踪.md`
- `templates/bootstrap/当前阶段Kanban.md`：用于生成 `当前阶段Kanban.md`
- `templates/bootstrap/当前状态快照.md`：用于生成 `当前状态快照.md`
- `templates/bootstrap/当前Hooks.default.md` / `templates/bootstrap/当前Hooks.git.md`：用于生成 `当前Hooks.md`

### 2.3 OpenCode 适配文档

- `agents/*.md`：OpenCode agent 定义模板
- `commands/*.md`：OpenCode command 定义模板
- `scripts/opencode.ps1`：一体化初始化 OpenCode 适配文件与项目运行文档

### 2.4 Claude Code 适配文档

- `claude/agents/*.md`：Claude Code agent 定义模板
- `claude/commands/*.md`：Claude Code command 定义模板
- `scripts/claude-code.ps1`：一体化初始化 Claude Code 适配文件与项目运行文档

## 3. 各角色读取顺序

### 3.1 Main Agent 默认读取顺序

1. `framework/MainAgent.md`
2. `framework/文档目录规范.md`
3. `framework/命令体系规范.md`
4. `framework/Kanban任务模板.md`
5. `framework/SubAgent任务下发模板.md`
6. 当前项目中的 `当前阶段Kanban.md`
7. 当前项目中的 `当前RoadMap.md`
8. 当前项目中的 `当前PRD.md`
9. 当前项目中的技术方案与项目约束文档

### 3.2 Dev Sub Agent 默认读取顺序

1. `framework/DevSubAgent.md`
2. Main Agent 下发的 Dev Task Package
3. 任务包中授权的文件路径对应上下文
4. 任务包中引用的接口契约与测试用例

### 3.3 QA Sub Agent 默认读取顺序

1. `framework/QASubAgent.md`
2. Main Agent 下发的 QA Task Package
3. 任务包中指定的改动文件与必要关联上下文
4. 任务包中引用的验收标准、风险说明与测试要求

---

## 4. 标准工作流

1. 人类提出 Idea 或问题
2. Main Agent 产出 `当前PRD.md`
3. Main Agent 产出 `当前RoadMap.md`
4. Main Agent 基于 RoadMap 产出 `当前阶段Kanban.md`
5. Main Agent 按 `framework/SubAgent任务下发模板.md` 派发 Dev / QA 任务包
6. Sub Agent 回传结果
7. Main Agent 更新 Kanban、Bug 追踪与状态文档

---

## 5. 最小必读集合

如果只想快速启动这套流程，最少应先读这 4 份文档：

1. `framework/总览入口.md`
2. `framework/MainAgent.md`
3. `framework/SubAgent任务下发模板.md`
4. `framework/Kanban任务模板.md`
