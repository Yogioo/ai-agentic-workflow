# 文档目录规范

## 1. 设计原则

- 文档按职责分层，不混放。
- 当前有效文档与历史归档文档分离。
- 文件名尽量稳定、可预测、可搜索。
- Main Agent 优先读取当前有效文档，而不是遍历历史文件。
- 面向人类的文档优先用户优化，面向系统的文档优先规范化与结构化。

---

## 1.1 文档表达分层

为避免“系统内部状态”直接污染用户体验，文档表达应按受众分层：

- 人类阅读优先文档：以易懂、自然、低门槛为首要目标
- 系统读取优先文档：以稳定、结构化、可解析、可同步为首要目标

推荐划分如下：

- 偏人类文档：`README.md`、交互式帮助输出、命令对用户的默认主输出
- 偏系统文档：`当前状态快照.md`、`当前PRD.md`、`当前RoadMap.md`、`当前阶段Kanban.md`、`BUG追踪.md`、任务包、回传结果

其中：

- `当前状态快照.md` 属于系统状态文档，不要求用户友好表达，优先保证状态一致性与字段稳定性
- `start` / `next` 的默认主输出属于人机交互层，必须优先用户友好表达
