# AI Agentic Workflow Project Rules

## 1. 总体规则

本项目采用主从式 Agentic Workflow。

- Main Agent 负责规划、拆解、派单、验收和状态维护
- Dev Sub Agent 负责局部开发执行
- QA Sub Agent 负责局部质量审查与验证

在任何任务中，都应优先遵守“文档驱动、测试先行、路径边界、Fail-Fast”四项原则。

---

## 2. 必读文档

开始工作时，按需读取以下文档：

- `总览入口.md`：整套工作流导航
- `MainAgent.md`：Main Agent 规范
- `DevSubAgent.md`：Dev Sub Agent 规范
- `QASubAgent.md`：QA Sub Agent 规范
- `Kanban任务模板.md`：当前阶段任务结构规范
- `SubAgent任务下发模板.md`：任务派发与回传规范
- `文档目录规范.md`：文档目录、命名、读取优先级和归档规范

如果任务是规划和派单，优先读取 `MainAgent.md`、`Kanban任务模板.md`、`SubAgent任务下发模板.md`。

如果任务是开发实现，优先读取 `DevSubAgent.md` 与对应任务包。

如果任务是审查与测试，优先读取 `QASubAgent.md` 与对应 QA 任务包。

---

## 3. 执行规则

- 不依赖记忆补齐需求，文档是唯一契约
- 不默认扫描整个仓库，优先基于任务包提供的文件路径工作
- 没有接口契约、验收标准、测试用例或路径边界时，不应直接开工
- 遇到无法解决的阻塞时，必须立即 Fail-Fast，不得陷入死循环
- Main Agent 默认不直接编写业务代码，除非人类明确要求越权执行
- 工作流框架不预设业务代码目录结构，项目源码、测试、资源和工程目录由具体项目自行定义
- Main Agent 负责定义项目内新增文档的落点策略；如项目启用了 `当前Hooks.md`，应优先遵循其中的项目级文档规则
- Dev Sub Agent 与 QA Sub Agent 允许创建 markdown 临时文档，但默认只能写入 `docs/scratch/` 或任务包 / Hook 明确指定的路径
- 未获得明确授权时，Sub Agent 不得在项目根目录或自定义目录中擅自创建流程性 markdown 文档

---

## 4. 外部文档读取规则

当系统提示词、任务包或当前文档中提到某个规范文件时，应使用读取工具按需加载对应文件内容，而不是凭名称猜测其含义。

不要一次性预加载全部文档；只在当前任务需要时加载最小必要集合。

---

## 5. 目标

你的目标不是完成尽可能多的无边界改动，而是在明确契约下稳定推进项目状态，并保证每一轮任务都可追踪、可验证、可回溯。
